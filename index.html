Core Python Concepts

BASIC
1
What is Python? What are its key features?
▶
✅ Answer
Python is a high-level, interpreted, general-purpose, dynamically typed programming language known for its clean and readable syntax. Key features:

  • Interpreted: Executed line by line; no compile step needed
  • Dynamically typed: No need to declare variable types
  • Object-Oriented: Supports classes, inheritance, polymorphism
  • Open Source & Cross-Platform: Runs on Windows, Linux, macOS
  • Extensive Libraries: NumPy, Pandas, Django, Flask, scikit-learn, etc.
  • Garbage Collection: Automatic memory management
  • Indentation-based: Uses whitespace instead of braces
2
Is Python compiled or interpreted?
▶
✅ Answer
Python is both compiled and interpreted. When you run a .py file, CPython first compiles it to bytecode (.pyc), then the Python Virtual Machine (PVM) interprets that bytecode line by line. PyPy uses JIT compilation for faster execution. Most people call Python "interpreted" because the compilation step is transparent to the user.
3
What is a dynamically typed language?
▶
✅ Answer
In a dynamically typed language, the type of a variable is determined at runtime, not at compile time. Python automatically detects the type based on the assigned value. 
x = 10       # x is int
x = "Hello"  # now x is str — no error!
Contrast this with statically typed languages like Java/C++ where you must declare the type explicitly.
4
What is the difference between a List, Tuple, Set, and Dictionary?
▶
✅ Answer
  • List []: Ordered, mutable, allows duplicates → [1, 2, 2, 3]
  • Tuple (): Ordered, immutable, allows duplicates → (1, 2, 2, 3)
  • Set {}: Unordered, mutable, no duplicates → {1, 2, 3}
  • Dictionary {}: Ordered (Python 3.7+), mutable, key-value pairs, unique keys → {"a": 1}
5
What is the difference between mutable and immutable objects?
▶
✅ Answer
Mutable: Objects that can be changed after creation → list, dict, set, bytearray
Immutable: Objects that cannot be changed after creation → int, float, str, tuple, frozenset, bool

When you "modify" an immutable object (like a string), Python creates a new object in memory rather than changing the existing one.

What are Python's built-in data types?
▶
✅ Answer
  • Numeric: int, float, complex, bool
  • Sequence: str, list, tuple, range
  • Mapping: dict
  • Set: set, frozenset
  • Binary: bytes, bytearray, memoryview
  • None: NoneType
7
What is the difference between / and // in Python?
▶
✅ Answer
/ → True division — always returns a float: 5/2 = 2.5
// → Floor division — returns the largest integer ≤ result: 5//2 = 2, -7//2 = -4
8
What is PEP 8?
▶
✅ Answer
PEP 8 is the Python Enhancement Proposal that provides style guidelines for writing Python code. Key rules include:
  • Use 4 spaces for indentation
  • Maximum line length of 79 characters
  • Use snake_case for variables/functions, PascalCase for classes
  • Two blank lines before class/function definitions
  • Imports on separate lines, grouped (stdlib, third-party, local)
9
What is the difference between == and is in Python?
▶
✅ Answer
== checks value equality (are the values the same?)
is checks identity equality (do they point to the same object in memory?) 
a = [1, 2]
b = [1, 2]
print(a == b)   # True  (same value)
print(a is b)   # False (different objects in memory)
10
What is None in Python?
▶
✅ Answer
None is Python's null value. It is the sole instance of the NoneType class. It represents the absence of a value. Functions that don't explicitly return a value return None by default. Always compare using is None, not == None.
11
What is variable scope? Explain LEGB rule.
▶
✅ Answer
Python resolves variable names using the LEGB rule (in this order):
  • Local — inside the current function
  • Enclosing — in enclosing function scopes (for closures)
  • Global — at module level
  • Built-in — Python's built-in names (len, print, etc.)
Use global keyword to modify a global variable inside a function, and nonlocal to modify an enclosing variable.

What are map(), filter(), and reduce()?

nums = [1, 2, 3]

15
What is a decorator in Python?
▶
✅ Answer
A decorator is a function that takes another function as input, adds some behaviour, and returns the modified function. Uses the @ syntax. 
def log_decorator(func):
    def wrapper(*args, **kwargs):
        print(f"Calling {func.__name__}")
        result = func(*args, **kwargs)
        print("Done")
        return result
    return wrapper

@log_decorator
def add(a, b):
    return a + b

add(3, 4)  # Calling add → 7 → Done
16
What is a generator in Python?
▶
✅ Answer
A generator is a function that uses yield instead of return. It lazily produces values one at a time, making it memory-efficient for large data. 
def count_up(n):
    for i in range(n):
        yield i

for val in count_up(3):
    print(val)  # 0, 1, 2
Unlike a list that stores all values in memory, a generator computes values on demand.

What is a list comprehension? What about dict/set comprehension?
▶
✅ Answer
Concise syntax to create collections: 
# List comprehension
squares = [x**2 for x in range(5)]

# With condition
evens = [x for x in range(10) if x % 2 == 0]

# Dict comprehension
d = {x: x**2 for x in range(4)}  # {0:0, 1:1, 2:4, 3:9}

# Set comprehension
s = {x % 3 for x in range(9)}  # {0, 1, 2}
18
What is the difference between shallow copy and deep copy?
▶
✅ Answer
Shallow copy: Creates a new object but references the same nested objects. Changes to nested items affect both. 
import copy
a = [[1, 2], [3, 4]]
b = copy.copy(a)
b[0].append(99)
print(a)  # [[1, 2, 99], [3, 4]] — a is affected!
Deep copy: Creates a completely independent copy, including all nested objects. 
c = copy.deepcopy(a)
c[0].append(100)
print(a)  # unchanged
19
What is pickling and unpickling?
▶
✅ Answer
Pickling is the process of converting a Python object into a byte stream (serialization) for storage or transfer. Unpickling is the reverse — reconstructing the object from bytes. 
import pickle
data = {"name": "Alice", "score": 95}
with open("data.pkl", "wb") as f:
    pickle.dump(data, f)
with open("data.pkl", "rb") as f:
    loaded = pickle.load(f)
20
What is the difference between range() and xrange()?
▶
✅ Answer
In Python 2: range() returns a list (all values in memory), xrange() returns a lazy iterator (memory efficient).

In Python 3: xrange() was removed. range() now behaves like the old xrange() — it's a lazy sequence object, not a list.

P3
Find the factorial of a number (iterative + recursive)
▶
✅ Solution
# Recursive
def factorial_r(n):
    if n <= 1: return 1
    return n * factorial_r(n - 1)

# Iterative
def factorial_i(n):
    result = 1
    for i in range(2, n+1):
        result *= i
    return result

print(factorial_r(5))  # 120
P4
Find duplicates in a list
▶
✅ Solution
def find_duplicates(lst):
    seen = set()
    dups = set()
    for item in lst:
        if item in seen:
            dups.add(item)
        seen.add(item)
    return list(dups)

print(find_duplicates([1,2,3,2,4,1]))  # [1, 2]

Flatten a nested list
▶
✅ Solution
# Using recursion (handles any depth)
def flatten(lst):
    result = []
    for item in lst:
        if isinstance(item, list):
            result.extend(flatten(item))
        else:
            result.append(item)
    return result

nested = [1, [2, [3, 4]], 5]
print(flatten(nested))  # [1, 2, 3, 4, 5]

P10
Write a Fibonacci sequence generator
▶
✅ Solution
# Generator (memory efficient)
def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        yield a
        a, b = b, a + b

print(list(fibonacci(8)))  # [0,1,1,2,3,5,8,13]

P12
Merge two sorted lists
▶
✅ Solution
def merge_sorted(l1, l2):
    result, i, j = [], 0, 0
    while i < len(l1) and j < len(l2):
        if l1[i] < l2[j]:
            result.append(l1[i]); i += 1
        else:
            result.append(l2[j]); j += 1
    result += l1[i:] + l2[j:]
    return result

print(merge_sorted([1,3,5], [2,4,6]))  # [1,2,3,4,5,6]

Object-Oriented Programming

INTERMEDIATE
O1
What are the 4 pillars of OOP?
▶
✅ Answer
  • Encapsulation: Bundling data and methods together; hiding internal state (using _ or __ prefix)
  • Inheritance: A class inheriting attributes/methods from a parent class
  • Polymorphism: Same interface, different implementations (method overriding)
  • Abstraction: Hiding complex implementation, exposing only necessary parts (abstract classes)
O2
What is __init__? What is self?
▶
✅ Answer
__init__ is the constructor method called automatically when a new object is created. self refers to the current instance of the class. 
class Dog:
    def __init__(self, name, breed):
        self.name = name
        self.breed = breed

    def bark(self):
        return f"{self.name} says Woof!"

d = Dog("Buddy", "Labrador")
print(d.bark())
O3
What is the difference between class method, static method, and instance method?
▶
✅ Answer
class MyClass:
    count = 0

    def instance_method(self):
        # Has access to instance (self) and class
        return self

    @classmethod
    def class_method(cls):
        # Has access to class (cls) but NOT instance
        return cls.count

    @staticmethod
    def static_method():
        # No access to class or instance — like a regular function
        return "I'm static"
O4
What is method overriding vs method overloading in Python?
▶
✅ Answer
Method Overriding: A child class redefines a method from the parent class. 
class Animal:
    def speak(self): return "..."

class Dog(Animal):
    def speak(self): return "Woof"  # overrides
Method Overloading: Python does not natively support it. The last definition wins. You simulate it using default args or *args.
O5
What is multiple inheritance? What is MRO?
▶
✅ Answer
Python supports multiple inheritance — a class can inherit from more than one parent. MRO (Method Resolution Order) is the order in which Python looks up methods, determined by the C3 Linearization algorithm. 
class A:
    def hello(self): print("A")

class B(A):
    def hello(self): print("B")

class C(A):
    def hello(self): print("C")

class D(B, C): pass

D().hello()  # B (follows MRO: D → B → C → A)
print(D.__mro__)  # Shows the full order
O6
What are dunder/magic methods?
▶
✅ Answer
Dunder (double underscore) methods define how objects behave with built-in operations: 
class Vector:
    def __init__(self, x, y):
        self.x, self.y = x, y

    def __add__(self, other):       # v1 + v2
        return Vector(self.x+other.x, self.y+other.y)

    def __repr__(self):             # repr(v)
        return f"Vector({self.x}, {self.y})"

    def __len__(self):              # len(v)
        return 2
Common ones: __str__, __repr__, __len__, __add__, __eq__, __lt__, __iter__, __getitem__, __contains__
O7
What is a property decorator?
▶
✅ Answer
@property allows you to access a method like an attribute, enabling controlled access to private data. 
class Circle:
    def __init__(self, radius):
        self._radius = radius

    @property
    def area(self):
        import math
        return math.pi * self._radius ** 2

c = Circle(5)
print(c.area)  # Called like attribute, not c.area()

Advanced Python Concepts

ADVANCED
A1
What is the GIL (Global Interpreter Lock)?
▶
✅ Answer
The GIL is a mutex in CPython that allows only one thread to execute Python bytecode at a time, even on multi-core systems. This prevents race conditions in memory management but limits true parallelism in CPU-bound multi-threaded programs.

Workarounds: Use multiprocessing (separate processes, no GIL sharing), or asyncio for I/O-bound tasks. Libraries like NumPy release the GIL for C-level operations.
A2
What is memory management in Python? How does garbage collection work?
▶
✅ Answer
Python uses reference counting as its primary GC mechanism. Every object has a ref_count; when it hits 0, memory is freed. Python also has a cyclic garbage collector (in the gc module) that handles circular references. 
import sys
a = []
print(sys.getrefcount(a))  # Shows reference count

import gc
gc.collect()  # Force cyclic GC
A3
What is a context manager and the with statement?
▶
✅ Answer
A context manager manages resource setup and cleanup using __enter__ and __exit__ methods. The with statement ensures resources are properly released even if an exception occurs. 
# Custom context manager
class ManagedFile:
    def __enter__(self):
        print("Opening")
        return self
    def __exit__(self, *args):
        print("Closing")

with ManagedFile() as f:
    print("Working")

# Using contextlib
from contextlib import contextmanager

@contextmanager
def timer():
    import time; start = time.time()
    yield
    print(f"Elapsed: {time.time()-start:.3f}s")
A4
What are iterators and iterables?
▶
✅ Answer
Iterable: Any object that implements __iter__() — returns an iterator. Examples: list, str, dict.
Iterator: Object that implements both __iter__() and __next__(). Maintains state between calls. 
class CountDown:
    def __init__(self, start): self.val = start
    def __iter__(self): return self
    def __next__(self):
        if self.val <= 0: raise StopIteration
        self.val -= 1; return self.val + 1

for n in CountDown(3): print(n)  # 3 2 1
A5
What is monkey patching?
▶
✅ Answer
Monkey patching is dynamically modifying a class or module at runtime, typically to extend or modify behavior without altering the original source code. 
class Dog:
    def bark(self): return "Woof"

def new_bark(self): return "WOOF WOOF!"

Dog.bark = new_bark  # monkey patch
print(Dog().bark())  # "WOOF WOOF!"
A6
What is metaclass in Python?
▶
✅ Answer
A metaclass is the class of a class — it defines how classes behave. Just as instances are created from classes, classes are created from metaclasses. The default metaclass is type. 
class Meta(type):
    def __new__(mcs, name, bases, namespace):
        print(f"Creating class: {name}")
        return super().__new__(mcs, name, bases, namespace)

class MyClass(metaclass=Meta):  # "Creating class: MyClass" printed
    pass
A7
What is asyncio? What are async/await?
▶
✅ Answer
asyncio is Python's built-in library for asynchronous I/O using an event loop. async def defines a coroutine, await suspends execution until an awaitable completes. 
import asyncio

async def fetch_data():
    print("Fetching...")
    await asyncio.sleep(1)   # non-blocking wait
    print("Done!")

asyncio.run(fetch_data())
Best for I/O-bound tasks (web requests, DB queries), not CPU-bound.
A8
What is the difference between threading and multiprocessing?
▶
✅ Answer
  • threading: Multiple threads within same process, share memory. Limited by GIL for CPU work. Best for I/O-bound tasks.
  • multiprocessing: Separate processes with own memory space. Bypasses GIL. Best for CPU-bound tasks.
  • asyncio: Single-threaded, event loop. Best for I/O-bound with many concurrent operations.

D1
Implement a stack using a list
▶
✅ Solution
class Stack:
    def __init__(self): self.items = []
    def push(self, x): self.items.append(x)
    def pop(self): return self.items.pop()
    def peek(self): return self.items[-1]
    def is_empty(self): return len(self.items) == 0

s = Stack()
s.push(1); s.push(2); s.push(3)
print(s.pop())  # 3
D2
Implement a queue using collections.deque
▶
✅ Solution
from collections import deque

q = deque()
q.append("a")    # enqueue
q.append("b")
q.popleft()       # dequeue → "a"
print(q)          # deque(['b'])
Use deque instead of list for queues — popleft() is O(1) vs O(n) for list.pop(0).
D3
Binary search implementation
▶
✅ Solution
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1  # not found

print(binary_search([1,3,5,7,9], 5))  # 2
D4
What is the time complexity of common list operations?
▶
✅ Answer
  • list.append(x): O(1) amortized
  • list[i]: O(1)
  • list.insert(i, x): O(n)
  • list.pop(): O(1)
  • list.pop(0): O(n)
  • x in list: O(n)
  • x in set/dict: O(1) average
  • dict[key]: O(1) average
  • sorted(list): O(n log n)
D5
How does a Python dictionary work internally?
▶
✅ Answer
Python dicts use a hash table. When you store a key-value pair, Python computes hash(key) to determine the bucket. On lookup, it computes the hash, finds the bucket, and checks key equality. This gives O(1) average time for get/set/delete. Keys must be hashable (immutable). Python 3.7+ maintains insertion order.

E2
How do you create a custom exception?
▶
✅ Answer
class InsufficientFundsError(Exception):
    def __init__(self, amount, balance):
        self.message = f"Cannot withdraw {amount}, balance is {balance}"
        super().__init__(self.message)

def withdraw(amount, balance):
    if amount > balance:
        raise InsufficientFundsError(amount, balance)
    return balance - amount

try:
    withdraw(100, 50)
except InsufficientFundsError as e:
    print(e)
